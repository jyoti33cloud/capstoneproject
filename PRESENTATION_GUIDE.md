## Speaker Notes
> Let me start with the problem. Autism Spectrum Disorder (ASD) is increasingly prevalent in Nepal, but the support system is severely lacking. **First**, there's limited awareness. Many parents don't know their child has autism until very late, and there's significant cultural stigma. **Second**, there are very few resources available. Nepal has only a handful of specialized therapists, mostly concentrated in Kathmandu. **Third**, parents feel isolated - there's no platform where they can connect with others going through similar experiences. **Fourth**, finding a qualified specialist is extremely difficult - there's no transparent verification system. And **finally**, most resources are in English, excluding a large portion of the Nepali population. This is where Asha comes in.

## Speaker Notes
> Here's how the architecture works. Users interact with a React application in their browser. When they perform actions - like logging in, creating a post, or booking an appointment - the frontend makes secure HTTPS requests to the Express.js backend. The backend processes these requests, handles authentication, validates data, and communicates with the PostgreSQL database. External services like Google OAuth handle authentication, SendGrid sends emails, and browser APIs provide text-to-speech. The entire application is version controlled with Git, hosted on GitHub, and automatically deployed to Render. This architecture ensures scalability, security, and reliability.

```
│                                                  │
│  AUTHORIZATION LEVELS                           │
│  ├─ User role (standard access)                 │

└──────────────────────────────────────────────────┘
```

## Speaker Notes
> Security is absolutely paramount in healthcare applications. We offer **two authentication methods** - traditional email/password and Google OAuth for convenience. Passwords are hashed using bcryptjs with 10 rounds of encryption - never stored in plain text. **API Security** includes HTTPS encryption on all requests, CORS protection, rate limiting to prevent abuse, and comprehensive input validation. **Data Protection** ensures user data is encrypted in transit and isolated per user. We have **three authorization levels**: regular users, specialists, and admins, each with appropriate permissions. **Tokens** expire after 7 days, and we handle logouts securely. The application is **GDPR-compliant**, meaning users have rights to their data, can export it, and can request deletion. This comprehensive security approach protects both parents' and children's sensitive data.
